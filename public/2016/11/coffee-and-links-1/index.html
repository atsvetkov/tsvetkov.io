Coffee and Links #1

Nov 12, 2016

Sometimes I find interesting things on the web, which don’t belong to a single category. Just random articles or news about technology. Since I still want to share these findings and because I normally do this kind of reading at the start of the day, I will be publishing this as a Coffee and links digest.

News

  • This autumn Microsoft has launched Windows Server 2016, with lots of improvements and new features, like containers support (Docker containers can now natively run on Windows) and a lightweight GUI-less version called Nano Server (small, fast, and perfect as a hosting platform).

  • Another thing from Microsoft: a new communication platform called Microsoft Teams, which seems to target the market where Slack dominates, but has an additional benefit of Office 365 integration. Slack was apparently so scared with this announcement that they posted a public letter to Microsoft, written in a condescending tone and reminding a famous welcoming letter from Apple to IBM published in Wall Street Journal in 1981. Some people think that Slack will regret doing this.

  • If you have ever passed any Microsoft certification exams, this one might be interesting for you: Microsoft has partnered with Acclaim to provide a digital badges system for displaying your certification status. Showing off has never been easier!

  • If you are still looking into where and how to start using .NET Core, Jon Hilton found a bunch of sample projects to look into and learn from (here’s part 1 and part 2). I am definitely going to check these out.

  • Lots of different databases and ORM frameworks already have support for .NET Core. Always interesting to see how people abuse (in a good way) different database systems: for example, Marten is a document database built on top of a relational one (PostgreSQL). Actually, in version 3.0 has Octopus Deploy migrated from RavenDB to SQL Server, but they creatively use it as a document storage too.

Tools

  • Puma is a security scanner for Visual Studio built on top of Roslyn, which can detect OWASP vulnerabilities by means of static code analysis. It can be installed into a single project as a NuGet package or added globally as a Visual Studio extension.

  • Intercooler.js is an interesting attempt to simplify working with AJAX requests from client-side code. What if you could define which endpoint to call from the markup itself? No viewmodels needed, just like this:

    <-- This anchor tag posts to '/click' when it is clicked -->
<a ic-post-to="/click"> Click Me! </a>

    It allows you to make GET/POST requests, automatically render returned response as HTML inside the corresponding tag, easily integrate with the rest of your JavaScript code and much more (check out the examples). Might be an interesting way of composing the elements of a web application into completely self-sufficient widgets. And it’s only 6.74KB gzipped.

Fun

  • Is there a single geek out there who has never played Pac-Man? If this brings up good memories from your childhood (it absolutely does for me), then you may want to read a history of the famous game and even some gameplay analysis.

Other

  • This is the best explanation of video compression I have ever read. It is not a trivial topic at all, but Sid Bala made it super easy to follow and understand with the help of carefully chosen words like entropy and mindfuck.
